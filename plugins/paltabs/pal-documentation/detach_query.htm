Removing an Attached Query from a Class

When a query has been attached to a class, you can break the attachment without removing the query from the knowledge base. You can do this in one of two ways:

  • In the Query Attachments pane in the PAL Queries Tab
  • By selecting the class at the Classes Tab

Note: When you remove a query's class attachment(s), any variable in the query statement that is not defined by a range will not be recognized by the PAL engine.

At the PAL Queries Tab

To remove a query from a class at the PAL Queries Tab:

  1. Go to the PAL Queries Tab and make sure the query whose attachment you want to remove is displayed. See Displaying Queries at the Queries Tab for more information.
  2. Select the query by clicking on it. The attached classes will be shown in the Query Attachments pane.
  3. Select the class the attachment to which you wish to remove.
  4. Click the Remove button at the right of the Attachments pane.
    The attachment to the class is removed, but the query and the class remain in the knowledge base. 

At the Classes Tab

To remove a query from a class at the Classes Tab:

  1. Go to the Classes Tab.
  2. Highlight the class from which you wish to remove the query.
  3. Highlight the query(s) you wish to remove in the SLOT-QUERY pane in the Class Form.
  4. Click the Remove button at the right of the SLOT-QUERY pane.
    The attachment to the class is broken, and the query no longer appears in the Class Form. However, the query remains in the knowledge base and can be viewed at the Instances Tab or at the PAL Queries Tab.

Note: If the SLOT-QUERY pane does not appear, you may first need to make the class queriable. See Making a Class Queriable for more information.
