Viewing a Query

You can view and edit an existing query using the PAL Expression Editor. To do this, you can open the editor in one of three ways:

  • Directly view a query at the PAL Queries Tab 
  • View the query as an instance of PAL-QUERY at the Instances Tab
  • If the query is attached to a class, you can view the attached query at the Classes Tab

Note: If the PAL Expression Editor does not display, or displays incorrectly, you may need to configure it. See Configuring the PAL Expression Editor for more information.

At the PAL Queries Tab

At the Queries tab, you may not see the query you wish to edit. In this case, you will have to make sure it is displayed.

  1. Go to the PAL Queries Tab
  2. In the Choose Queries pane, click on the query you wish to edit. 
    OR
  3. If the query you wish to edit is not visible in the Choose Queries pane, click the  Add button to view the list of queries in the knowledge base, then choose the one you want and click OK.
  4. Click the View button at the upper right of the Choose Queries pane.
    The editor opens in a separate window.

At the Instances Tab

  1. Go to the Instances Tab.
  2. Display the classes under :SYSTEM-CLASS, display the classes under QUERY, and click on :PAL-QUERY to highlight it. 
  3. Go to the Direct Instances pane and click the View button.
    The PAL Expression Editor appears in the Forms pane of the Instances tab.

At the Classes Tab

To view a query that is attached to a class at the Classes Tab:

  1. Go to the Classes Tab.
  2. Highlight the class to which the query is attached.
    The Class Form for the selected class is displayed at the right of the Classes Tab.
  3. Select the query you wish to edit in the SLOT-QUERY pane in the Class Form.
  4. Click the View button at the upper right of the SLOT-QUERY pane. 
    The PAL Expression Editor for the selected query opens in a separate window.

Note: If the SLOT-QUERY pane does not display, you may need to make your class queriable. See Making a Class Queriable for more information.
